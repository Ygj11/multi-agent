# P0.6 删除 Graph 顶层 Cache 字段专项任务

## 任务定位

本任务位于 `P0.5-graph-state-slimming-task.md` 之后。

P0.5 已经完成：

- `GRAPH_STATE_FIELD_AUTHORITY` 覆盖所有 `AgentGraphState` 字段。
- 每个字段都有 `owner / source / kind`。
- `selected_skill_*` 顶层字段已标记为 `cache`，权威来源是 `subagent_result`。
- `approval_request` 顶层字段已标记为 `snapshot`，权威来源是 `ApprovalStore`。
- 审批恢复已经优先从 `ApprovalStore.get(approval_id)` 读取详情。
- E2E skill selection 测试已经改为读取 `state["subagent_result"]["selected_skill_id"]`。

本任务目标是进入下一阶段：**真正删除 Graph 顶层 cache/snapshot 字段**，让 Graph State 更轻、更清晰，避免后续节点继续依赖冗余字段。

## 本次要删除或弱化的字段

### 1. 必删：顶层 selected skill cache

从 `AgentGraphState` 顶层删除：

```text
selected_skill_id
selected_skill_metadata
skill_selection_score
skill_selection_reason
```

权威来源改为：

```text
state["subagent_result"]["selected_skill_id"]
state["subagent_result"]["selected_skill_metadata"]
state["subagent_result"]["skill_selection_score"]
state["subagent_result"]["skill_selection_reason"]
```

注意：这只删除 **Graph 顶层 cache 字段**，不要删除：

- `SubAgentResult.selected_skill_id`
- `SubAgentResult.selected_skill_metadata`
- `SubAgentResult.skill_selection_score`
- `SubAgentResult.skill_selection_reason`
- `SubAgentContext.selected_skill_id`
- `SubAgentContext.selected_skill_metadata`
- `SubAgentContext.skill_selection_score`
- `SubAgentContext.skill_selection_reason`

子 Agent 内部仍需要这些字段表达执行结果。

### 2. 推荐删除：顶层 approval_request snapshot

从 `AgentGraphState` 顶层删除：

```text
approval_request
```

权威来源改为：

```text
approval_id -> ApprovalStore.get(approval_id)
```

Graph 顶层长期只保留审批摘要字段：

```text
approval_required
approval_id
approval_status
parent_approval_id
root_approval_id
current_approval_id
next_approval_id
approval_depth
manual_intervention_required
approval_submit_result
pending_messages
pending_tools
pending_tool_call
```

注意：不要删除 `ApprovalRequest` schema，不要删除 `ApprovalStore`，不要删除审批表字段。

### 3. 可选删除：target_subagent

`target_subagent` 当前在 `GRAPH_STATE_FIELD_AUTHORITY` 中标记为 `deprecated`。如果搜索确认没有业务消费点，可以从 `AgentGraphState` 中删除。

## 风险点

### 1. Approval resume 不能只靠 graph state

删除 `approval_request` 后，所有 resume 详情必须从 `ApprovalStore` 读取。

如果 callback 时 `approval_id` 丢失，应直接报错，不要尝试用旧 snapshot 兜底。

### 2. LangGraph checkpoint 中旧 state 可能仍含旧字段

本地旧 checkpoint 里可能保留旧字段。

删除 TypedDict 字段后，LangGraph 传入旧字段通常不会直接报错，因为 `TypedDict` 运行时不强制校验；但业务代码不要再读取旧字段。

如需生产迁移，后续可以增加 checkpoint state sanitizer。

### 3. 不要误删 SubAgentResult 字段

本任务只删除 Graph 顶层 cache，不删除子 Agent 执行结果字段。

### 4. 不要为了测试保留顶层 cache

如果测试失败，优先改测试断言读取权威来源，而不是把顶层 cache 加回来。
