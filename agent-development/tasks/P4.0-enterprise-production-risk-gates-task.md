# P4.0 Enterprise Production Risk Gates Task

优先级：企业级 P0

## 任务定位

本任务用于收口生产环境最基础的安全边界。

当前项目支持 mock/real 工具模式、InternalLLMProvider 本地 fallback、OpenSDK LLM、外部审批系统等能力。这些能力对本地开发非常有用，但如果在 staging/prod 环境被误用，会造成非常严重的误判：

- 看起来调用成功，实际调用的是 mock 工具。
- 看起来 LLM 可用，实际走的是本地 fallback。
- 看起来审批提交成功，实际打到了 mock approval URL。
- 看起来有真实用户身份，实际走的是 body fallback identity。

企业级系统必须先做环境 gate：开发环境可以宽松，生产环境必须强校验。

## 当前问题

### 1. 缺少统一 `APP_ENV`

当前 settings 有多个独立开关：

- `POS_TOOL_MODE`
- `TROUBLESHOOTING_TOOL_MODE`
- `ENABLE_OPENSDK_LLM`
- `ENABLE_REAL_LLM`
- `INTERNAL_LLM_API_URL`
- `APPROVAL_SYSTEM_URL`
- `ALLOW_REQUEST_BODY_IDENTITY_FALLBACK`

但没有一个总环境语义告诉系统当前是 local、test、staging 还是 prod。

### 2. 生产环境可能误用 mock tool

当前工具模式允许：

```text
POS_TOOL_MODE=mock
TROUBLESHOOTING_TOOL_MODE=mock
```

如果上环境时忘记改成 real，系统仍可启动。

### 3. InternalLLMProvider 可能走本地 fallback

`InternalLLMProvider` 在没有 `base_url` 时会走 `_local_fallback_response`。

这在本地测试方便，但生产环境必须禁止。

### 4. 外部审批默认 URL 带 mock 语义

当前默认：

```text
APPROVAL_SYSTEM_URL=http://mock-approval-system.local/api/approval/requests
```

生产环境必须禁止默认 mock URL。

### 5. 身份 body fallback 不适合生产默认开启

`ALLOW_REQUEST_BODY_IDENTITY_FALLBACK=true` 对本地调试友好，但生产环境应依赖可信网关、JWT 或服务端身份。

## 架构目标

新增统一环境配置：

```text
APP_ENV=local|test|staging|prod
```

生产约束：

```text
APP_ENV=prod:
  POS_TOOL_MODE must be real
  TROUBLESHOOTING_TOOL_MODE must be real
  InternalLLMProvider must have INTERNAL_LLM_API_URL if selected
  OpenSDKLLMProvider must have OPENAI_API_KEY and OPENAI_BASE_URL if selected
  APPROVAL_SYSTEM_URL must not contain mock
  ALLOW_REQUEST_BODY_IDENTITY_FALLBACK must be false
```

staging 约束建议接近 prod，但可允许显式例外：

```text
APP_ENV=staging:
  mock tool 默认禁止
  如需联调 mock，必须显式 ENABLE_STAGING_MOCKS=true
```

local/test 保持开发友好。

## 实施步骤

### 1. 扩展 Settings

在 `Settings` 中新增：

```python
app_env: Literal["local", "test", "staging", "prod"] = "local"
enable_staging_mocks: bool = False
```

在 `get_settings()` 中读取：

```text
APP_ENV
ENABLE_STAGING_MOCKS
```

### 2. 增加环境枚举校验

如果 `APP_ENV` 不是合法值，启动直接报错。

### 3. 增加生产 gate 函数

建议在 `app/main.py` 或新文件 `app/config/environment.py` 中增加：

```python
def validate_runtime_environment(settings: Settings) -> None:
    ...
```

该函数在 `create_app()` 早期调用。

### 4. 禁止生产 mock tools

规则：

```text
APP_ENV=prod and POS_TOOL_MODE=mock -> ValueError
APP_ENV=prod and TROUBLESHOOTING_TOOL_MODE=mock -> ValueError
APP_ENV=staging and *_TOOL_MODE=mock and ENABLE_STAGING_MOCKS=false -> ValueError
```

### 5. 禁止生产 LLM 本地 fallback

如果最终使用 `InternalLLMProvider`，则：

```text
APP_ENV=prod and INTERNAL_LLM_API_URL empty -> ValueError
```

如果使用 `OpenSDKLLMProvider`，则：

```text
APP_ENV=prod and OPENAI_API_KEY empty -> ValueError
APP_ENV=prod and OPENAI_BASE_URL empty -> ValueError
```

### 6. 禁止生产 mock approval

规则：

```text
APP_ENV=prod and "mock" in APPROVAL_SYSTEM_URL.lower() -> ValueError
```

### 7. 禁止生产 body fallback identity

规则：

```text
APP_ENV=prod and ALLOW_REQUEST_BODY_IDENTITY_FALLBACK=true -> ValueError
```

### 8. 更新 `.env.example`

增加明确分区：

```text
# Runtime environment
APP_ENV=local
ENABLE_STAGING_MOCKS=false
```

并标注生产建议配置。

### 9. 增加测试

新增测试覆盖：

- local 允许 mock。
- prod 禁止 POS mock。
- prod 禁止 troubleshooting mock。
- prod 禁止 InternalLLMProvider 无 base_url。
- prod 禁止 mock approval URL。
- prod 禁止 body fallback identity。
- staging 默认禁止 mock。
- staging + ENABLE_STAGING_MOCKS=true 允许 mock。

## 不在本任务范围

- 不删除 mock 工具实现。
- 不改 tool handler 业务逻辑。
- 不做 ToolContract。
- 不做 prompt eval。
- 不做 observability 指标。

## 验收标准

- `APP_ENV=prod` 时 mock 工具配置会导致启动失败。
- `APP_ENV=prod` 时无真实 LLM 配置会导致启动失败。
- `APP_ENV=prod` 时 mock approval URL 会导致启动失败。
- `APP_ENV=prod` 时 body identity fallback 会导致启动失败。
- `.env.example` 明确说明 local/staging/prod 配置差异。
- 新增测试全部通过。
