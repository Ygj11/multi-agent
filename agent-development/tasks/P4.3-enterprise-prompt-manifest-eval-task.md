# P4.3 Enterprise Prompt Manifest And Evaluation Task

优先级：企业级 P2

## 任务定位

本任务用于建立 prompt 生命周期治理和 LLM 行为评测体系。

当前项目已经有多类 prompt：

- query rewrite
- intent recognition
- agent selection
- skill selection
- subagent reasoning
- memory summary
- final compliance

这些 prompt 已经开始承担关键决策，但缺少企业级治理能力：

- 没有版本。
- 没有 manifest。
- 没有绑定输出 schema。
- 没有绑定 eval suite。
- 改 prompt 后无法系统评估影响。
- LLM 输出解析仍偏宽松。

本任务目标是让 prompt 从“文本模板”升级为“可版本化、可评测、可回滚的生产资产”。

## 当前问题

### 1. Prompt 无版本

当前 prompt 文件修改后，运行日志无法回答：

- 用的是哪个 prompt 版本？
- 哪次 prompt 改动导致结果变化？
- 当前线上 prompt 和本地 prompt 是否一致？

### 2. Prompt 无 manifest

现在无法从一个统一文件看出：

- scene 对应哪个 system/user prompt
- 输出 schema 是什么
- 评测集是什么
- 适用模型是什么
- 是否允许 tools

### 3. LLM 输出解析偏宽松

当前主要通过 `parse_json_object()` 获取 dict，再手动取字段。

企业级更适合：

- strict Pydantic model
- parse error code
- schema version
- invalid output diagnostics

### 4. 缺少 eval cases

当前 pytest 覆盖较多，但不是面向真实 LLM 决策质量的 eval。

缺少：

- query rewrite 多轮评测
- intent taxonomy 评测
- agent selection 评测
- skill selection 评测
- tool calling 评测
- final answer 评测

## 实施步骤

### 1. 建立 PromptManifest

新增 `app/prompts/manifest.yaml`：

```yaml
version: "1.0.0"
scenes:
  query_rewrite:
    version: "2026-06-11.1"
    system: query_rewrite/system.md
    user: query_rewrite/user.md
    output_schema: QueryRewriteLLMOutput
    eval_suite: query_rewrite_multiturn_v1
    tools_allowed: false
  intent_recognition:
    version: "2026-06-11.1"
    system: intent_recognition/system.md
    user: intent_recognition/user.md
    output_schema: IntentRecognitionLLMOutput
    eval_suite: intent_taxonomy_v1
    tools_allowed: false
```

覆盖所有 scene。

### 2. 扩展 PromptLoader

`PromptLoader` 支持：

- 读取 manifest。
- 根据 scene 获取 prompt paths。
- 返回 prompt version。
- 校验 prompt 文件存在。

### 3. 定义 LLM strict output schema

新增 schema：

- `QueryRewriteLLMOutput`
- `IntentRecognitionLLMOutput`
- `AgentSelectionLLMOutput`
- `SkillSelectionLLMOutput`

使用 Pydantic strict model。

要求：

- JSON parse 失败和 schema validation 失败要区分。
- 非法枚举值要有明确错误。
- 缺少必填字段要有明确错误。

### 4. 替换裸 dict 解析

把关键节点中类似：

```python
data = parse_json_object(response.content)
```

升级为：

```python
parsed = parse_llm_json_schema(response.content, QueryRewriteLLMOutput)
```

返回：

```text
success
data
error_code
error_detail
```

### 5. prompt version 写入 decision trace

关键节点结果记录：

```text
prompt_scene
prompt_version
output_schema
model
finish_reason
parse_status
```

### 6. 建立 eval cases

新增评测 case 文件。

query rewrite case 示例：

```yaml
suite: query_rewrite_multiturn_v1
cases:
  - id: clarification_reply_endo_apply_seq
    conversation:
      - role: user
        content: 保全任务完成，保单9200100000458846为什么没更新？
      - role: assistant
        content: 还缺少 apply_seq、endorseType，请补充后我再继续处理。
        metadata:
          need_clarification: true
          missing_required_entities: [apply_seq, endorseType]
          entities:
            policy_no: "9200100000458846"
    current_query: "930021042875719，001028"
    expected:
      rewrite_type: clarification_reply
      entities:
        policy_no: "9200100000458846"
        apply_seq: "930021042875719"
        endorseType: "001028"
      need_clarification: false
```

### 7. 建立 eval runner

新增脚本：

```bash
uv run python scripts/run_prompt_evals.py --suite query_rewrite_multiturn_v1
```

runner 支持：

- fake deterministic provider
- real provider
- 输出 summary report
- 输出失败 case id

### 8. 增加测试

测试覆盖：

- manifest 可加载。
- manifest 中 prompt 文件存在。
- manifest 中 schema 名存在。
- 每个 scene 有 version。
- eval cases 可加载。
- strict schema 能识别 invalid JSON。
- strict schema 能识别缺字段。

## 不在本任务范围

- 不要求 CI 直接调用真实 LLM。
- 不做前端 eval dashboard。
- 不重写所有 prompt 内容。
- 不做 ToolContract。

## 验收标准

- `app/prompts/manifest.yaml` 存在并覆盖所有 LLM scene。
- 每次关键 LLM 调用可记录 prompt version。
- LLM 输出 schema validation 失败时有明确错误码。
- eval cases 能被 runner 加载并执行。
- prompt 文件缺失或 schema 缺失时测试失败。
- 新增测试全部通过。
