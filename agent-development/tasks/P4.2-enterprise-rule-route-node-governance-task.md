# P4.2 Enterprise Rule Route Node Governance Task

优先级：企业级 P1

## 任务定位

本任务用于治理当前散落在代码中的规则、阈值、路由条件和节点契约。

当前项目的 LangGraph 主流程已经比较完整，但很多企业级治理点仍以 Python if/else、硬编码 tuple、固定阈值、固定话术的形式存在。短期可以跑，长期会导致：

- 新增业务关键词必须改代码。
- 路由条件分散，容易漏分支。
- 节点输入输出没有契约，新增字段难治理。
- fallback 文案和 retry 策略散落在节点里。
- AgentCard 打分权重和 Skill 打分权重治理方式不一致。

本任务目标是把规则和路由从“代码逻辑”升级为“可测试的策略层”。

## 当前问题

### 1. LangGraph node 缺少声明式契约

当前 `AgentGraphState` 有字段权威表，但每个 node 没有独立声明：

- 需要哪些输入字段
- 产出哪些字段
- 可能返回哪些 route
- 可能产生哪些 error code
- 哪些字段是 runtime only

### 2. Route policy 分散在 `graph.py`

当前 route 函数包括：

- `entry_route`
- `clarification_route`
- `human_approval_route`
- `after_create_approval_route`
- `compliance_route`

这些函数目前比较简单，但随着流程复杂，会让 `graph.py` 越来越像策略中心。

### 3. Query Rewrite 规则硬编码

当前硬编码内容包括：

- `EXPLICIT_REFERENCE_SIGNALS`
- `WEAK_FOLLOW_UP_SIGNALS`
- `STRONG_ANCHOR_ENTITY_TYPES`
- `ENTITY_TYPE_ALIASES`
- `ORDINAL_TARGETS`

### 4. Intent fallback 规则硬编码

当前 fallback 关键词写在 `IntentRecognitionNode` 中，例如：

- POS 查询关键词
- troubleshooting 关键词
- sub_intent 关键词
- unknown 澄清话术

### 5. AgentCard scoring 权重硬编码

`AgentCardLoader.match_candidates()` 中的评分权重写死在代码里。

Skill 评分已经有 `app/skills/scoring_policy.yaml`，Agent 评分也应该同等治理。

## 架构目标

新增四类治理对象：

```text
NodeContract
  每个 LangGraph node 的输入、输出、路由和错误码契约

RoutePolicy
  clarification、approval、verification 等 graph route 规则

QueryContextReferencePolicy
  多轮上下文继承和引用判断规则

IntentFallbackPolicy / AgentRoutingPolicy
  意图 fallback 和 AgentCard scoring 策略
```

## 实施步骤

### 1. 建立 NodeContract

新增模型：

```python
class NodeContract(BaseModel):
    node_name: str
    required_inputs: list[str]
    optional_inputs: list[str] = []
    outputs: list[str]
    routes: list[str] = []
    failure_codes: list[str] = []
```

为当前节点建立契约：

- `route_entry`
- `load_session`
- `save_user_message`
- `query_rewrite`
- `intent_recognition`
- `build_orchestrator_context`
- `discover_agents`
- `select_agent`
- `assemble_task`
- `dispatch_agent`
- `build_clarification_answer`
- `check_human_approval_required`
- `create_approval_request`
- `submit_approval_request`
- `pause_for_approval`
- `pre_answer_verify`
- `regenerate_compliant_answer`
- `fallback_answer`
- `save_assistant_message`
- `compress_short_memory`
- `finalize_response`

### 2. 增加 NodeContract 测试

测试内容：

- 所有 `graph.add_node()` 的 node 都有 contract。
- contract 中 outputs 必须存在于 `AgentGraphState` 或允许的动态字段白名单。
- route 返回值必须在 contract.routes 中。
- failure_codes 必须来自统一 failure code 枚举。

### 3. 建立 RoutePolicy

把 route 判断集中到 `app/runtime/route_policy.py`。

建议函数：

```python
route_entry(state) -> Literal["resume", "normal"]
route_clarification(state) -> Literal["clarify", "continue"]
route_approval_required(state) -> Literal["required", "not_required"]
route_after_create_approval(state) -> Literal["submit", "manual"]
route_verification(state) -> Literal["passed", "retry", "fallback"]
```

`graph.py` 只调用 RoutePolicy，不直接写条件。

### 4. 外置 Query Context Reference Policy

新增 `app/query/context_reference_policy.yaml`：

```yaml
version: "1.0.0"
explicit_reference_signals:
  - 这个
  - 那个
  - 刚才
weak_follow_up_signals:
  - 为什么
  - 怎么办
strong_anchor_entity_types:
  - policy_no
  - apply_seq
  - request_id
entity_type_aliases:
  policyNo: policy_no
  applySeq: apply_seq
ordinal_targets:
  第一个: 0
  第二个: 1
short_query_without_anchor_max_len: 16
```

`QueryRewriteNode` 从 policy loader 读取，不再在代码中维护这些常量。

### 5. 外置 Intent Fallback Policy

新增 `app/query/intent_fallback_policy.yaml`：

```yaml
version: "1.0.0"
intents:
  pos_query:
    confidence: 0.86
    keywords:
      - 保全实时查询
      - 可做保全项
    entity_hints:
      - customer_no
    sub_intents:
      pos_available_items:
        keywords:
          - 可做保全项
  troubleshooting:
    confidence: 0.90
    keywords:
      - 排查
      - 失败
      - 异常
    entity_hints:
      - request_id
      - error_code
unknown:
  confidence: 0.42
  clarification_question: 请补充你要办理的业务类型，例如排查或保全实时查询。
```

`IntentRecognitionNode` 只负责调用 policy。

### 6. Agent Routing Policy YAML 化

新增 `app/agents/routing_policy.yaml`：

```yaml
version: "1.0.0"
thresholds:
  rule_confident_threshold: 8.0
  rule_margin_threshold: 3.0
  top_k: 3
weights:
  intent_match: 6.0
  intent_capability_keyword: 2.0
  sub_intent_match: 2.0
  required_entity_present: 2.5
  required_entity_missing: -1.5
  optional_entity_present: 1.2
  capability_keyword: 1.5
  query_keyword: 1.0
  enabled: 0.5
```

`AgentCardLoader.match_candidates()` 使用 policy 权重。

### 7. Policy version 写入 decision trace

Query Rewrite、Intent、Agent Selection 的结果中记录：

```text
policy_version
policy_name
```

## 不在本任务范围

- 不做 PromptManifest。
- 不做 ToolContract。
- 不做真实 LLM eval。
- 不改变业务意图 taxonomy 内容。
- 不改变工具实现。

## 验收标准

- 所有 LangGraph node 都有 NodeContract。
- RoutePolicy 单测覆盖所有 route。
- Query Rewrite 不再直接维护引用信号常量。
- Intent fallback 关键词不再硬编码在 Python 中。
- AgentCard scoring 权重来自 YAML。
- policy loader 有版本字段。
- 规则变更可以通过改 YAML + 单测完成。
