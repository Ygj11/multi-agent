# P0.5 Graph State 字段瘦身专项任务

## 任务定位

本任务位于 `P0-task.md` 完成之后、`P1-task.md` 开始之前。

P0 已经完成主链路瘦身、ToolExecutor pipeline、Graph handlers、bootstrap 拆分、mock/production 边界收口，并新增了 `GRAPH_STATE_FIELD_AUTHORITY` 字段权威表。当前剩余问题是：Graph state 中仍保留了一些兼容性顶层字段，它们方便调试和旧测试，但会让节点间输入输出继续膨胀。

本任务目标是做一次低风险、可回滚的 Graph State 字段消费点迁移和冗余字段删除。

## 为什么要做

当前 state 中存在一些重复表达：

```text
selected_skill_id / subagent_result.selected_skill_id
selected_skill_metadata / subagent_result.selected_skill_metadata
skill_selection_score / subagent_result.skill_selection_score
skill_selection_reason / subagent_result.skill_selection_reason
verification_results / pre_answer_verification_result
approval_request / ApprovalStore approval record
available_agents / AgentCardLoader.list_available_agents()
principal / auth_context.principal
entities / entity_bag / conversation_window
```

这些字段并不全是错误。有些是兼容 cache，有些是 debug trace，有些是派生上下文快照。但如果不明确迁移，后续新增功能会继续随手读写顶层字段，Graph State 会越来越胖。

## 当前已完成前置

- `app/runtime/graph_state.py::GRAPH_STATE_FIELD_AUTHORITY` 已声明关键字段权威来源。
- `tests/test_graph_state_authority.py` 已覆盖核心字段。

- `/api/chat`、审批、verification、memory 主链路已有契约测试保护。

## 建议执行步骤

### 1. 更新字段权威表

检查 `GRAPH_STATE_FIELD_AUTHORITY` 是否覆盖当前 `AgentGraphState` 中所有字段。

为每个字段补充：

```text
owner
source
kind: authoritative | cache | debug | snapshot | deprecated
delete_after: 可选
```

验收：

- `tests/test_graph_state_authority.py` 覆盖所有 `AgentGraphState` 字段。
- 每个字段都能说明“谁是权威来源”。

### 2. 迁移 selected_skill 顶层字段消费点

当前顶层字段：

```text
selected_skill_id
selected_skill_metadata
skill_selection_score
skill_selection_reason
```

权威来源应为：

```text
subagent_result.selected_skill_id
subagent_result.selected_skill_metadata
subagent_result.skill_selection_score
subagent_result.skill_selection_reason
```

执行：

- 搜索所有 `state["selected_skill_id"]` / `state.get("selected_skill_id")`。
- 后续节点优先从 `subagent_result` 读取。
- approval resume 中如果需要这些字段，优先从 `subagent_result` 或 pending state 中读取。
- 修改测试，不再要求顶层 `state["selected_skill_id"]` 是主契约。

保留策略：

- 第一轮可以保留顶层字段作为兼容 cache。
- 第二轮确认测试稳定后，再从 `AgentGraphState` TypedDict 和 `dispatch_agent` 返回值中删除。

验收：

- `tests/test_skill_selection_end_to_end.py` 改为优先断言 `state["subagent_result"]["selected_skill_id"]`。
- `/api/chat` 返回结构不变。

### 3. 弱化 verification_results

权威路由字段：

```text
pre_answer_verification_result
```

`verification_results` 只作为 trace/debug 列表。

执行：

- Graph 路由只允许读取 `pre_answer_verification_result`。
- ResponseAdapter 不读取 `verification_results`。
- MessageStore metadata 如需记录，只记录 `pre_answer_verification_result` 摘要。

验收：

- `pre_answer_verify` 后 `pre_answer_verification_result` 必须存在。
- `verification_results` 不作为任何路由条件。
- `tests/test_final_compliance_check.py` 继续通过。

### 4. 弱化 approval_request 顶层详情

权威详情来源：

```text
ApprovalStore
```

Graph 顶层长期只保留：

```text
approval_required
approval_id
approval_status
parent_approval_id
root_approval_id
approval_depth
current_approval_id
next_approval_id
manual_intervention_required
```

执行：

- 检查哪些节点必须读取 `state["approval_request"]`。
- 对 callback/resume 场景，优先通过 `ApprovalStore.get(approval_id)` 读取详情。
- pending response 不依赖完整 `approval_request`。

保留策略：

- 第一轮仍允许 `approval_request` 作为 resume snapshot。
- 如果 LangGraph checkpoint 恢复链路稳定，再考虑删除顶层完整 approval_request。

验收：

- `tests/test_approval_full_flow.py`
- `tests/test_approval_callback_approved.py`
- `tests/test_approval_callback_rejected.py`
- `tests/test_approval_chain_resume.py`
- `tests/test_approval_idempotency.py`

全部通过。

### 5. 标记 available_agents 为 debug 字段

`available_agents` 当前用于 trace/debug，后续节点不应依赖它作为权威输入。

执行：

- 保持 `discover_agents` 节点输出用于 graph_path/debug。
- `select_agent` 继续通过 `AgentCardLoader / AgentSelectionNode` 获取候选，不从 `state["available_agents"]` 取权威数据。
- 测试中如果只验证存在，可保留；不要把它作为业务选择依据。

验收：

- `tests/test_langgraph_flow.py` 继续通过。
- `tests/test_architecture_acceptance.py` 继续通过。

### 6. 明确 principal 与 auth_context 的关系

权威身份字段：

```text
auth_context
```

`principal` 是为了 ToolExecutor、VerificationService、Agent access 快速读取保留的展开 cache。

执行：

- `GRAPH_STATE_FIELD_AUTHORITY` 明确 `principal` 是 cache。
- 后续新增节点优先接收 `auth_context`，需要 Principal 时再展开。
- 不要把 body identity 当成可信身份覆盖 header/gateway 身份。

验收：

- auth 相关测试继续通过。
- ToolExecutor 权限测试继续通过。
